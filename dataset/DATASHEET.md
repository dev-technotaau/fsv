# Datasheet — Ninja Fence Segmentation Dataset v1.0

_Following the "Datasheet for Datasets" framework (Gebru et al., 2021)._

---

## Motivation

**For what purpose was the dataset created?**
To train fence-segmentation models for the **Ninja Fence Staining Visualizer**, a browser-deployed tool that lets homeowners upload a photo of their cedar fence and visualize the appearance of various stain colors applied to it. The models need to detect wood-fence pixels in a user's photo and generate a binary mask for downstream color-overlay rendering.

**Who created the dataset?**
Built by the Technotau engineering team (2026) for a client-deliverable ML product. Not a public benchmark.

**Who funded the creation?**
Client project; internal R&D budget.

---

## Composition

**What does each instance represent?**
A single JPEG image of either:
- A **wood fence** (positive class — the model should detect and mask these pixels), or
- A **non-fence scene or fence-like-but-not-wood-fence structure** (negative class — the model should NOT generate a mask)

Each instance in `manifest.jsonl` is one JSON row with UUID, file path, class, subcategory, source attribution, dimensions, SHA-256, dhash, original URL, and (if applicable) Vision-verified labels.

**How many instances?**
**33,423 total** (post-deduplication, post-Vision-QA):
- 21,414 positive (wood fences — cedar, redwood, pine, etc.)
- 12,009 negative (pure-random scenes, non-wood fences, fence-like-non-fence structures)

**Is the dataset a sample of a larger set?**
Yes — curated from ~60k raw scrape candidates. Filters applied:
- Resolution: width ≥ 800, height ≥ 600
- File size: 40 KB – 25 MB
- Aspect ratio: 0.4 – 2.5
- Google Vision accept at threshold 0.4-0.6 (fence-family labels)
- Cross-set SHA dedup (251 removed)
- Near-dup dhash dedup within each scrape run (Hamming ≤ 5)
- Post-hoc Vision QA (336 non-wood-fence images moved to rejected/)

**What data does each instance consist of?**
- JPEG file (typically 40KB–5MB, mostly 200KB–1MB)
- Width × Height between 800 × 600 and 8000 × 8000
- Metadata (class, subcategory, source, original URL, Vision label, SHA-256, dhash, resolution tier)

**Are there labels/targets?**
Image-level: yes (`class`: pos/neg, `subcategory`: one of 32 fine-grained types).
Pixel-level masks: **NOT YET** — masks generated by Gemini Vision auto-label pass + human QA (see `ANNOTATION_GUIDELINES.md`).

**Is any information missing?**
Not systematically. Occasional `origin_page`, `title`, or `extra.license` fields are null for sources that don't provide them.

**Are there recommended data splits?**
Yes — `dataset/splits/`:
- `train.jsonl` — 23,397 rows (70.0%)
- `val.jsonl` — 5,011 rows (14.99%)
- `test.jsonl` — 5,015 rows (15.0%, **read-only locked**)

Split is **stratified by (class × subcategory)** and **dhash-group-aware** (near-duplicates always co-located). Reproducible via seed=42. See `dataset/splits/split_info.json`.

High-resolution (≥1024 shorter edge) subsets also available:
- `manifest_hq.jsonl`, `splits/{train,val,test}_hq.jsonl` — 20,700 total

**Are there errors, sources of noise, or redundancies?**
- **Estimated label noise**: ~1-3% of positives may be borderline cases (e.g., lattice-top fences where the lattice portion is ambiguous). Expected to be caught during human mask QA.
- **Source bias**: heavy skew toward U.S. residential cedar fences (matches deployment market, intentional).
- **Subcategory imbalance**: `neg:neg_pure_random` is 8,985 (27% of dataset) vs. some `pos:urban_rundown` at only 6 rows. Flagged during prepare_dataset; training should use class-balanced sampling or loss weighting.

## Collection Process

**How was the data acquired?**
Automated multi-source scraping over April 2026 using a custom pipeline (`data_scraper/` in this repo). Sources:
- **API-clean**: Pexels, Unsplash, Pixabay, Wikimedia Commons
- **API-authenticated (disabled)**: Reddit (OAuth not configured)
- **Web-scraped (Playwright)**: Google Images, Bing Images, Pinterest, Houzz
- **Direct gallery scrapes**: 30 curated fence-company websites (verified live)

Query generation:
- 139 hand-written fence queries covering wood types, styles, occlusion scenarios, weather, lighting, and scale variations
- 22 client-specific cedar-focused queries
- 150 Gemini-generated expansions (prompt engineered for hard-positive scenarios)
- 191 curated hard-positive queries targeting edge cases (motion blur, extreme angles, reflections, etc.)
- 226 curated negative queries across 10 hard-negative categories

**Was the data collection process validated?**
Yes — each scraped image passed:
1. **Content filter** (URL/title blocklist for stock-photo watermarks)
2. **Quality filter** (dimensions, file size, aspect)
3. **SHA deduplication** (exact-match)
4. **dhash near-dup deduplication** (Hamming ≤ 5 via BK-tree)
5. **Google Vision acceptance** (fence-family labels at confidence ≥ 0.4-0.6)

**Does the dataset relate to people?**
Mostly no (fence photos). A small fraction (`pos:humans_animals` = 57 rows) contain identifiable people. See `PII_SCAN_REPORT.md` if Priority-6 PII scan has been run.

## Preprocessing / Cleaning / Labeling

**Was any preprocessing done?**
- All images **re-encoded to JPEG quality 92** at scrape time
- **EXIF metadata stripped** during save
- Color mode normalized to **RGB**
- `width`/`height`/`bytes`/`sha256` verified from disk (not trusted from metadata)

**Was "raw" data saved?**
No — only the processed JPEGs are retained. Original URLs are preserved in `origin_url` field for each row, so a re-scrape is always possible.

**Is software to preprocess/clean/label available?**
Yes, all in this repo:
- `data_scraper/` — scraping pipeline
- `prepare_dataset.py` — integrity check + cross-dedup + subcategory classification
- `split_dataset.py` — stratified split with dhash-group awareness
- `tools/audit_licenses.py` — source licensing audit
- `tools/select_golden_set.py` — benchmark subset selector
- `tools/standardize_images.py` — resolution tiering + HQ subset
- `tools/scan_pii.py` — face-detection scan (when Priority 6 completed)

## Uses

**For what tasks has the dataset been used?**
Currently: **segmentation model pretraining** (5 architectures: UNet++, DeepLabV3, SegFormer B0/B5, Mask2Former, YOLOv8-seg). Will be used for deployed cedar-staining visualizer.

**Is there a repository of papers/systems using the dataset?**
No — internal client deliverable.

**What other tasks could it be used for?**
- Generic fence detection (not cedar-specific)
- Wood-structure segmentation broadly (with mask relabeling)
- Domain adaptation research (fine-grained material segmentation)

**Is there anything that should NOT be done with this dataset?**
- **Do not use Risky-tier sources commercially without legal clearance** (see `LICENSE_AUDIT.md`). For commercial deployment, filter to `manifest_safe.jsonl` first.
- **Do not publish or redistribute** the scraped images — licensing is per-source and bulk redistribution may violate terms.
- **Do not use for face recognition or identifying individuals** — any incidentally-photographed people should be treated as subjects of the image, not labeled targets.
